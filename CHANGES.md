## Authentication Changes

### Before (User/Password)
```yaml
environment:
  - EMERCOIN_USER=${EMERCOIN_RPC_USER:-rpcuser}
  - EMERCOIN_PASS=${EMERCOIN_RPC_PASS:-rpcpassword}
```

### After (Cookie-based)
```yaml
environment:
  - EMERCOIN_COOKIE_FILE=/data/.cookie
volumes:
  - emercoin-data:/data:ro
```

## Benefits

1. **Security**: Cookie-based authentication is more secure than hardcoded credentials
2. **Simplicity**: No need to manage RPC username/password in environment variables
3. **Consistency**: All services now use the same authentication method
4. **Cleaner Stack**: Removed unused AmneziaWG services that were cluttering the configuration

## Next Steps

To start the stack with the new configuration:

```bash
# Validate the configuration
docker compose config --quiet

# Start the services
docker compose up -d

# Check service status
docker compose ps

# View logs
docker compose logs -f
```

## Notes

- The `.cookie` file is automatically generated by Emercoin core on startup
- Services that need Emercoin RPC access now mount the `emercoin-data` volume as read-only
- All AmneziaWG-related ports (51820, 51821, 8002) are now freed up
- User interfaces now work with both `localhost` and custom domains
