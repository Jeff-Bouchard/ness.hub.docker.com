───────────────────────── SECURITY POSTURE SUMMARY ──────────────────────────────────

 

WHAT IS EXPOSED WHY

───────────────────────── ──────────────────────────────────────────────────

:443/udp WireGuard mesh entry point — encrypted by design

:42424/udp LBRTYnet peerd peer bootstrap — UDP, no auth surface

:6661 Emercoin P2P blockchain sync — public P2P protocol

:2222 SSH pubkey+NVS only, Ed25519, brute-force throttled

:8000 Skywire visor UI — recommend firewall to VPN clients only

:1053 DNS/NVS resolver for local clients

 

WHAT IS LOCKED DOWN ATTACK PREVENTED

───────────────────────── ──────────────────────────────────────────────────

RPC ports → 127.0.0.1 remote RPC exploitation, credential theft

Admin RPC → 127.0.0.1 VPN server takeover

I2P/Yggdrasil → 127.0.0.1 admin interface exposure

pyuheprng/tools → 127.0.0.1 entropy service abuse, internal API scraping

cap_drop:ALL everywhere kernel capability exploits, container escape

no-new-privileges all setuid escalation after initial compromise

read_only FS + noexec tmpfs malware persistence, dropper execution

ICC disabled on bridge lateral movement between containers via bridge

:? hard-fail on passwords accidental deployment with default credentials

SSH: no TCP forwarding pivot / SOCKS proxy into internal namespace

SSH: MaxAuthTries 3 brute force, credential stuffing

SSH: AEAD+ETM ciphers only BEAST, Lucky13, padding oracle, compression oracle

SSH: Ed25519 only RSA factoring, DSA nonce reuse, NIST curve concerns
